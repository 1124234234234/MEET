# 会议室智能合规分析系统 - API接口文档

## 基本信息

- **服务地址**: `http://{host}:5000`
- **API版本**: v1.0
- **数据格式**: JSON
- **字符编码**: UTF-8

## 二、会议管理

### 2.1 创建会议（上传音频分析）

**POST** `/api/meetings`

**Content-Type**: `multipart/form-data`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| audio_file | File | 是 | 音频文件（支持 mp3, wav, ogg, flac） |
| meeting_title | String | 否 | 会议标题，默认"未命名会议" |
| enable_diarization | Boolean | 否 | 是否启用说话人分离，默认false |
| enable_compliance | Boolean | 否 | 是否启用合规检查，默认true |

**响应**:
```json
{
  "code": 200,
  "message": "分析已开始",
  "meeting_id": 123
}
```

### 2.2 获取会议列表

**GET** `/api/meetings`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| page_size | Integer | 否 | 每页数量，默认10 |
| status | String | 否 | 筛选状态（processing/finished/failed） |

**响应**:
```json
{
  "code": 200,
  "data": [
    {
      "id": 123,
      "title": "产品需求评审会",
      "date": "2026-07-20 10:30:00",
      "duration": 1800,
      "status": "finished",
      "summary": "会议讨论了Q3产品规划...",
      "total_score": 85.5,
      "score_level": "合格"
    }
  ],
  "total": 50,
  "page": 1,
  "page_size": 10
}
```

## 三、实时语音转写（WebSocket）

### 3.1 连接地址

**WebSocket**: `ws://{host}:5000/socket.io/?EIO=4&transport=websocket`

### 3.2 事件列表

| 事件名 | 方向 | 说明 |
|--------|------|------|
| `connect` | 客户端→服务端 | 建立连接 |
| `start_transcription` | 客户端→服务端 | 开始实时转写 |
| `audio_chunk` | 客户端→服务端 | 发送音频块 |
| `stop_transcription` | 客户端→服务端 | 停止转写 |
| `transcription_result` | 服务端→客户端 | 实时转写结果 |
| `transcription_stopped` | 服务端→客户端 | 转写停止通知 |
| `realtime_analysis_progress` | 服务端→客户端 | 分析进度更新 |
| `transcription_final` | 服务端→客户端 | 最终分析结果 |
| `error` | 服务端→客户端 | 错误信息 |

### 3.3 消息格式

#### 3.3.1 开始转写

**客户端发送**:
```json
{
  "language": "zh",
  "sample_rate": 16000
}
```

**参数说明**:
- `language`: 语言代码（zh/en/ja/ko/fr/de/es/ru/ar/pt），默认zh
- `sample_rate`: 采样率，必须为16000

#### 3.3.2 发送音频块

**客户端发送**:
```json
{
  "audio_data": "base64编码的PCM音频数据"
}
```

**注意事项**:
- 音频格式：PCM格式，16位，单声道，采样率16000Hz
- 数据编码：Base64编码
- 建议每500ms发送一块（约16KB）

#### 3.3.3 停止转写

**客户端发送**:
```json
{}
```

#### 3.3.4 实时转写结果（服务端推送）

**服务端推送**:
```json
{
  "text": "你好，今天的会议内容是...",
  "is_final": false,
  "timestamp": 1234567890
}
```

**参数说明**:
- `text`: 转写文本
- `is_final`: 是否为最终结果（true表示完整句子，false表示临时结果）
- `timestamp`: Unix时间戳

#### 3.3.5 转写停止通知（服务端推送）

**服务端推送**:
```json
{
  "audio_file": "/uploads/xxx.wav",
  "audio_duration": 30.5,
  "text": "完整转写文本..."
}
```

#### 3.3.6 分析进度（服务端推送）

**服务端推送**:
```json
{
  "progress": 45,
  "message": "正在进行说话人分离..."
}
```

#### 3.3.7 最终分析结果（服务端推送）

**服务端推送**:
```json
{
  "meeting_id": 123,
  "text": "完整转写文本...",
  "summary": "会议摘要...",
  "keywords": ["关键词1", "关键词2"],
  "topics": ["主题1", "主题2"],
  "sentiment": {"score": 0.75, "label": "positive"},
  "compliance_report": {
    "total_score": 85.5,
    "score_level": "合格",
    "risk_keywords": [],
    "suggestions": ["会议内容合规"]
  },
  "duration": 30
}
```

## 四、知识库管理

### 4.1 获取知识库列表

**GET** `/api/knowledge-base`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| page_size | Integer | 否 | 每页数量，默认10 |
| item_type | String | 否 | 类型筛选（policy/risk_keywords/key_points） |

**响应**:
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "title": "理财产品销售合规管理办法",
      "content": "理财产品销售必须遵守合规要求...",
      "item_type": "policy",
      "keywords": ["风险", "销售", "投资者"],
      "required_points": [],
      "status": "active"
    }
  ],
  "total": 10
}
```

### 4.2 创建知识库条目

**POST** `/api/knowledge-base`

**请求体**:
```json
{
  "title": "新政策标题",
  "content": "政策内容描述...",
  "item_type": "policy",
  "keywords": ["关键词1", "关键词2"],
  "required_points": ["要点1", "要点2"]
}
```

**参数说明**:
- `item_type`: policy（政策）/ risk_keywords（风险关键词）/ key_points（必传要点）

**响应**:
```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 123,
    "title": "新政策标题",
    "status": "active"
  }
}
```

### 4.3 上传知识库文件

**POST** `/api/knowledge-base`

**Content-Type**: `multipart/form-data`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 政策文件（.txt/.pdf/.docx/.doc） |
| item_type | String | 否 | 类型，默认policy |

**响应**:
```json
{
  "code": 200,
  "message": "上传成功",
  "data": {...},
  "parsed": {
    "title": "解析出的标题",
    "keywords": ["关键词1", "关键词2"],
    "required_points": ["要点1"],
    "content_length": 2000
  }
}
```

### 4.4 获取单个知识库条目

**GET** `/api/knowledge-base/{item_id}`

**响应**:
```json
{
  "code": 200,
  "data": {...}
}
```

### 4.5 更新知识库条目

**PUT** `/api/knowledge-base/{item_id}`

**请求体**:
```json

  "title": "更新后的标题",
```

**响应**:

### 4.6 删除知识库条目
