AI语音识别与翻译系统 实验说明书

实验目的：
1. 了解基于OpenAI Whisper的语音识别技术原理
2. 掌握RESTful API服务的开发与部署方法
3. 熟悉Python Flask框架的使用
4. 掌握音频文件处理与语言检测技术

一、项目概述

本项目是一个基于OpenAI Whisper的语音识别API服务，支持自动语言检测，并可将识别结果翻译成目标语言。

技术栈：
- 后端：Python 3.10+ / Flask 3.1+
- 语音识别：OpenAI Whisper (medium模型)
- 翻译：Helsinki-NLP/opus-mt (transformers)
- 前端：HTML5 / JavaScript
- 音频处理：FFmpeg

功能特性：
- 自动检测音频语言（支持中、英、日、韩、法、德、西、俄、阿、葡等10+种语言）
- 支持中文与英文互译
- 提供RESTful API接口，方便集成到其他系统
- 内置Web Demo页面，上传音频即可试用

二、环境要求

硬件要求：
- CPU：多核处理器（推荐Intel i5及以上）
- 内存：至少8GB（Whisper medium模型约占4GB）
- 磁盘：至少10GB可用空间（模型文件约2GB）

软件要求：
- Python 3.10+
- FFmpeg（音频处理工具）
- Git（可选，用于克隆仓库）

三、安装与启动

步骤1：检查Python版本
打开终端，执行命令：
python --version
确保显示 Python 3.10 或更高版本。

步骤2：安装依赖包
进入项目目录 voice-reco，执行命令：
pip install openai-whisper flask flask-cors transformers torch -i https://pypi.tuna.tsinghua.edu.cn/simple

步骤3：检查FFmpeg
执行命令：
ffmpeg -version
确保显示FFmpeg版本信息。

步骤4：启动服务
执行命令：
python app.py

启动成功后会显示：
==================================================
  AI Voice Recognition & Translation API
==================================================
Initializing Whisper model...
Whisper medium model loaded successfully

Ready to accept requests on 0.0.0.0:5000
 * Running on http://127.0.0.1:5000

步骤5：验证服务
打开浏览器访问 http://localhost:5000，或在终端执行：
Invoke-RestMethod http://localhost:5000/health
预期返回：{"status":"ok","message":"API service is running"}

五、API接口说明

接口1：健康检查
地址：GET http://localhost:5000/health
功能：检查服务是否正常运行
请求参数：无
响应示例：
{"status":"ok","message":"API service is running"}

接口2：获取支持语言
地址：GET http://localhost:5000/languages
功能：获取系统支持的所有语言列表
请求参数：无
响应示例：
{"zh":"Chinese","en":"English","ja":"Japanese","ko":"Korean",...}

接口3：语音识别
地址：POST http://localhost:5000/transcribe
功能：上传音频文件进行语音识别，可选翻译
请求参数：
- audio：音频文件（必填，支持wav、mp3、ogg等格式）
- target_language：目标语言代码（可选，如en、zh）
响应示例：
{
  "transcribed_text": "你好，世界",
  "detected_language": "zh",
  "language_name": "Chinese",
  "translated_text": "Hello, world",
  "target_language": "en"
}

七、常见问题与解决方法

问题1：服务启动时报错"out of memory"
解决方法：
- 关闭其他占用内存的程序
- 将模型改为small或base（占用内存更小）

问题2：音频文件无法识别
解决方法：
- 确保音频格式为wav、mp3或ogg
- 确保音频文件没有损坏
- 尝试使用其他音频文件测试

问题3：翻译功能不工作
解决方法：
- 确认网络连接正常（首次使用需要下载翻译模型）
- 目前仅支持中文↔英文互译

问题4：服务无法启动
解决方法：
- 检查端口5000是否被其他程序占用
- 执行 netstat -ano | findstr :5000 查看占用情况
- 使用 taskkill /F /PID <进程号> 关闭占用进程
